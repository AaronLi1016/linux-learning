# 应用编程概念

应用编程是指在操作系统之上编写用户态程序. Linux 应用程序通常通过 C 库函数或系统调用访问内核能力, 例如文件, 进程, 网络和设备.

## 1. 为什么需要系统调用

Linux 运行时可按职责分为三层:

1. 应用程序: 运行在用户态, 不能直接访问硬件和内核数据.
2. 系统调用接口: 用户态进入内核态的受控入口.
3. 内核: 管理进程, 内存, 文件系统, 设备驱动和网络协议栈.

系统调用的作用:

- 为应用程序提供访问内核服务的接口.
- 隔离用户态和内核态, 提高系统稳定性和安全性.
- 统一不同硬件和驱动的访问方式.

常见系统调用包括:

- `open`: 打开文件或设备.
- `read`: 读取数据.
- `write`: 写入数据.
- `close`: 关闭文件描述符.
- `fork`: 创建进程.
- `execve`: 执行新程序.

在 Linux 中, 设备通常以文件形式暴露给应用层, 例如 `/dev/xxx`. 应用程序调用 `open`, `read`, `write`, `close` 时, 内核会根据文件对象找到对应驱动的 `file_operations` 结构体, 再调用驱动实现的函数. 常见成员包括 `open`, `read`, `write`, `release`. 这实现了应用逻辑和底层驱动的解耦.

## 2. 库函数是什么

库函数是由库提供的函数接口. Linux 应用开发中常见的是 C 标准库和 POSIX 库函数, 例如 `printf`, `malloc`, `fopen`, `pthread_create`.

库函数和系统调用不同:

- 库函数运行在用户态, 由库实现.
- 系统调用会进入内核态, 由内核完成操作.
- 有些库函数会封装系统调用, 例如 `fopen` 通常会间接使用 `open`.

C 库文件通常位于 `/lib`, `/lib64`, `/usr/lib` 或 `/usr/lib64`. 具体位置和发行版, 架构有关.

## 3. 应用程序的 main 函数

Linux C 程序通常从 `main` 函数开始执行. 常见写法如下:

```c
int main(int argc, char **argv)
{
    return 0;
}
```

参数含义:

- `argc`: 命令行参数数量.
- `argv`: 命令行参数字符串数组.
- `argv[0]`: 程序名或执行路径.
- `argv[1]` 到 `argv[argc - 1]`: 用户传入的参数.

示例:

```bash
./test 55
```

此时:

- `argc` 等于 `2`.
- `argv[0]` 通常是 `./test`.
- `argv[1]` 是 `55`.

注意, `argv[argc]` 的值是 `NULL`, 可用于判断参数数组结束.

## 4. 文件描述符

文件描述符是内核为进程维护的整数句柄. 应用程序通过文件描述符操作文件, 设备, 管道和 socket.

默认文件描述符:

- `0`: 标准输入, `stdin`.
- `1`: 标准输出, `stdout`.
- `2`: 标准错误, `stderr`.

例如, `open` 成功后返回一个非负文件描述符, 失败时返回 `-1`, 并设置 `errno` 表示错误原因.

## 5. 错误处理

Linux C 程序应检查系统调用和库函数返回值. 常见规则:

- 返回 `-1` 通常表示系统调用失败.
- 返回 `NULL` 通常表示指针类库函数失败.
- `errno` 保存最近一次错误码.
- `perror` 可打印错误说明.

示例:

```c
int fd = open("/dev/xxx", O_RDONLY);
if (fd < 0) {
    perror("open");
    return 1;
}
```
